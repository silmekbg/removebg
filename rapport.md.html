Rapport sur une application web de suppression de fond d’image avec Streamlit et Python

https://github.com/silmekbg/removebg

Introduction

L’objectif de ce projet est de créer une application web permettant de supprimer facilement le fond d’une image. Pour ce faire, j’ai choisi d’utiliser Streamlit, un framework Python pour la création d’applications web interactives, et la bibliothèque rembg pour le traitement d’images.

Démarche

  1. Recherche de solutions existantes: Nous avons examiné les solutions existantes pour la suppression de fond d’image et avons identifié rembg comme la bibliothèque la plus appropriée pour notre cas d’utilisation, en raison de sa précision et de sa facilité d’utilisation.

  2. Conception de l’architecture de l’application: En utilisant Streamlit, nous avons conçu l’architecture de l’application en définissant les différentes pages et composants nécessaires pour permettre aux utilisateurs de télécharger une image, de la traiter et de visualiser le résultat ; utiliser Streamlit permet également de pouvoir bénéficier de l’application partout grâce à une WebUI (responsive)

  3. Implémentation: Nous avons procédé à l’implémentation de l’application en suivant l’architecture définie, en utilisant Python pour le back-end. L’intégration de rembg a été réalisée pour le traitement d’images en arrière-plan grâce à python.

Écueils rencontrés

  1. Performance: L’un des principaux défis rencontrés était d’assurer des performances optimales lors du chargement de la page, qui est malencontreusement long. J’ai ensuite choisi de convertir chaque image uploader en PNG.

  2. Gestion des erreurs: La gestion des erreurs était également un défi, en particulier lorsqu’il s’agissait de gérer les différentes libraires nécessaires pour le fonctionnement de l’application. Il fallait donc être en mesure de pouvoir gérer les conflits entre les différentes versions de python.

Solutions retenues

  1. Optimisation du code: Nous avons optimisé notre code en utilisant des bibliothèques Python efficaces.

  2. Gestion des erreurs améliorée: Nous avons amélioré la gestion des erreurs en mettant en place des validations côté client pour les images téléchargées et en fournissant des messages d’erreur clairs et des instructions pour aider les utilisateurs à résoudre les problèmes rencontrés.

  3. Gestion des conflits: J’ai mis en place un environnement virtuel (virualenv) afin d’éviter tous conflits avec d’autres projets présent sur mon pc. C’est ce qui a réglé les écueils rencontrés. Je souhaitai en premier lieu utiliser poetry dans le but de gérer les dépendances python, mais finalement opter pour un outil classique (virtualenv + requirements.txt). J’utiliserai poetry, sûrement, pour un futur projet.

Ce rapport résume la démarche, les défis rencontrés et les solutions retenues pour développer une application web de suppression de fond d’image avec Streamlit et Python.
